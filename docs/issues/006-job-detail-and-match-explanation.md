# Issue 006: Job Detail And Match Explanation

Type: AFK

Blocked by: Issue 005

User stories covered: 5, 6, 15, 16, 17, 26

## What to build

Create the Job Detail view that explains why ResumeAgent selected a role and what it plans to do next. The view should include job overview, job description, requirements, match analysis, resume selection, application history, and portal notes.

The purpose is to make automated job selection understandable before the user approves resume tailoring.

## Acceptance criteria

- [ ] Job Detail shows title, company, location, portal, URL, description, and requirements.
- [ ] Match analysis includes matched skills, weak or missing skills, blocked terms, target title match, and score reasoning.
- [ ] The region-based resume choice is shown clearly.
- [ ] Existing application history is shown if the job matches a previous application.
- [ ] The user can generate or regenerate a tailored resume from the detail view.
- [ ] The user can skip the job with a reason.
